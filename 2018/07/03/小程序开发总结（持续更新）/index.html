小程序开发总结（持续更新）

小程序开发学习过程中的一些总结：

  1. scroll-view 标签需要在内部再套一层块级标签，如 view ，并设置 container 足够宽

  2. iPhone 和安卓机显示样式不同，需要在 JS 中判断设备类型并针对性解决，如商品促销标签

  3. 不应该频繁使用 setData ，不应该传入大量数据，否则会影响性能

  4. 判断是否到达底部，进行数据加载，可以使用 scroll-view 的 bindscrolltolower 属性

  5. 使用 template 不要忘记传入 data

  6. 指定 ajax 的先后顺序，把后执行的放在先执行的 success 中

  7. line-height 需要比字号大一些，避免整行文字显示不完整，多行文字（多用于商品标题），行高比字号大一点，安卓设备大 3-4rpx，iOS设备大 1-2rpx，文字块的高度，比两倍（或多倍）行高的和小 2rpx

  8. 列表第二列商品标签显示不全问题（如图），原因尚不明确，可能与微信 rpx 计算方式有关系，暂时将外部 card 宽度从 50% 改为定宽解决，应该仍有其他解决方法

  9. wx.getSystemInfo API 非常有用，可以判断平台等等设备信息，微信开发者工具调试中不能模拟，需要真机调试

  10. （部分?）安卓手机不支持 rgba 格式的透明背景

  11. （基础）判断平台并修改样式的时候要注意 CSS 选择器优先级的问题，多个标签比少标签的优先级要高

  12. 商品标签在 iPhone 6/7 和 iPhone 6p/7p 不同尺寸屏幕上，内部文字不垂直居中，可以适当增加1-2rpx 行高

  13. 一些 API 如 createSelectorQuery 是异步操作的，若执行在如 setData 比较大量的数据写入之后时，可能会因为 setData 有性能延迟而发生错误

  14. 小程序内嵌 web-view 刷新方法：小程序自带的原生API只能用于刷新小程序页面，不能刷新 web-view 页面。

    • 最好的方法：使用 wx:if 给 web-view 绑定一个 boolean 变量，利用 if 会刷新 dom 节点的性质，更新 boolean（先false再true）会使 web-view 页面刷新。

      问题：目前没有发现问题

    • 不完美的方法：在需要刷新页面的时候，先将 url 变量变成空字符串，再重新赋值。

      问题：在网页模拟器上，后退会出现url为空时候的错误页，但在iOS真机上正常

    • 不要使用的方法：web-view 页面会在 url 改变的时候刷新，可以通过更改url的方式（如尾部增加一个时间戳作为参数）使页面刷新。

      问题：iOS系统中每次后退时会回到上一个页面，多次跳转时要点击多次后退，使用体验差

  15. 收集 formid 时若需要在内容外包一层 button 标签，应该清除这个 button 的默认样式，避免内部样式错乱

  16. 如何禁止页面滚动：

    不用下拉刷新的方法：

    • 整个布局用<scroll-view>作为根节点，绑定 scroll-view 的 scroll-y 属性<scroll-view scroll-y="false">
    • 样式文件中设置 Page 的 overflow-y 属性值为hidden
    • 样式文件中设置 scroll-view 的 height 属性值为100%

    使用下拉刷新的方法：

    • 在外层添加一个view：<view class="page" style="height: ; overflow: ;"> 然后动态算出设备高度

  17. 【不只是小程序的问题】：JavaScript会自动取消数值类型后面无意义的0，导致解析JSON之后数值位数不一致，建议转换成字符串，或者在数据较少的时候统一toFixed
