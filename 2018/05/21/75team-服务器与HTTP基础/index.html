  10. HTTP 状态码

    1开头：服务器中间状态，如准备资源

    2开头：200正常返回

    3开头：状态跳转，301资源永久迁移，302临时迁移，304请求资源自上次到现在未改变，直接从缓存返回

    4开头：403没有权限，404资源不存在，405动作错误

    5开头：服务器内部错误

  16. Cache-Control 缓存策略

    • 又称强缓存，普通刷新会忽略它，但并不会清除它，需要强制刷新
    • 浏览器强制刷新，请求会带上 Cache-Control: no-cache（或max-age=0）和 Pragma: no-cache 头

    所以在浏览器上点击刷新按钮，得到的就是最新数据，但从另一个链接转到的就不会

    解决服务器缓存问题：

    • 在资源文件后面加一个随机数可以立即更新，避免缓存后不更新的bug

  17. 协商缓存

    • 又称弱缓存，普通刷新会启动弱缓存，忽略强缓存
    • 只有从地址栏或收藏夹输入网址、通过链接引用资源等情况下，浏览器才会启用强缓存
    • 这也是为什么有时候我们更新一张图片、一个js文件，页面依然是旧的，但世界浏览器访问那个图片或文件，看到的内容却是新的

    方法：派一个“最后修改时间 last modified”的时间戳，之后通过发送 HTTP请求让服务端比较时间戳

    图10

  18. 协商缓存：etag，不派 last modified 而派 etag

    checksum 给文件算签名

  19. HTTP Upgrade（WebSocket）

    判断是一个 request 还是 upgrade，如果是后者就走 websocket 协议，建立真正的 WebSocket 连接，如果是前者就使用正常的 HTTP 协议

    编写 WebSocket 服务器，传输的是一个 buffer 之后按照规则进行解码（它实际上是 HTTP 的一个升级）实际使用其实很少自己使用和实现

    使用 Node 中的库：ws、socket.io等
