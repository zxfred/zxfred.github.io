小程序踩坑总结 - 业务相关

遇事不决三板斧：

  • 重启开发者工具
  • ​
  1. 奇怪的 inajax 问题，红包页用特定账号会请求两次，initPage 里面增加 inajax 判断后解决
  2. loading 可以写在 onload 生命周期里，toast 需要写在 onshow 里面，因为切换页面时会产生一些bug
  3. 一般页面请求多个异步接口，避免闪动都会做 hack
  4. 接口正确性判断只需要判断 errorCode 不用判断 errorMsg
  5. 他人新建开发分支需要 git pull 一下本机才能显示
  6. 收集 formid 时如果需要在内容外包一层 button 标签，应该清除这个 button 的默认样式，避免内部样式错乱
  7. 统一后端处理的数据
    • （英文原版书）是否显示原价
    • 评分（会处理成整数或带0.5的小数）
  8. 测试方面如果出现代码不是最新的情况，删除所有代码，重新更新
  9. 坑：某 ajax 请求（首页feed）如果接口错误，在开发者工具上返回200，但在真机上返回302，不会执行 success 和 fail，只会执行 complete
