React Native 环境搭建

##平台

开发平台：macOS

App平台：Android / iOS

Tips：建议不要用 create-react-native-app 和 expo 搭建，国内用户可能产生网络问题，无法创建新项目（至少我遇到了）

##环境搭建

  1. 安装node

    • brew install node

  2. 安装 react-native-cli

    • npm i -g react-native-cli

  3. 安装 Java JDK 版本 8 或以上

  4. 安装 Android Studio

    • Android SDK
    • Android SDK Platform
    • Performance (Intel ® HAXM)
    • Android Virtual Device

  5. 安装安卓 SDK - Android 6.0 (Marshmallow)

    • Google APIs
    • Android SDK Platform 23
    • Intel x86 Atom_64 System Image
    • Google APIs Intel x86 Atom_64 System Image

  6. SDK Tools

    • 23.0.1

  7. 配置环境变量（使用其他 shell 需要改相应的 shell 配置）

    • $HOME/.bash_profile
    • export ANDROID_HOME=$HOME/Library/Android/sdk
    • export PATH=$PATH:$ANDROID_HOME/tools
    • export PATH=$PATH:$ANDROID_HOME/tools/bin
    • export PATH=$PATH:$ANDROID_HOME/platform-tools

    使用 echo $PATH 和 echo $ANDROID_HOME 检测

  8. 建立 React Native 程序

    react-native init myProject

  9. 使用 Android Studio 建立一个安卓手机模拟器

    使用 adb devices 查看模拟器

  10. 运行项目

    cd myProject
    react-native run-android

问题

###真机上运行时可能会遇到白屏的情况

找到并开启悬浮窗权限

###Android 5.0及以上(API 21+)的安卓设备上

设备连接USB数据线到电脑，并开启USB调试，运行adb reverse tcp:8081 tcp:8081

Android 5.0以下，通过Wi-Fi连接本地开发服务器

  1. 确保你的电脑和手机设备在同一个Wi-Fi环境下。
  2. 在设备上运行你的React Native应用。
  3. 你应该会看到一个“红屏”错误提示。这是正常的，下面的步骤会解决这个报错
  4. 摇晃设备，或者运行adb shell input keyevent 82，可以打开开发者菜单
  5. 点击进入Dev Settings
  6. 点击Debug server host for device
  7. 输入你电脑的IP地址和端口号（譬如10.0.1.1:8081）。在Mac上，你可以在系统设置/网络里找查询你的IP地址。在Windows上，打开命令提示符并输入ipconfig来查询你的IP地址。在Linux上你可以在终端中输入ifconfig来查询你的IP地址。
  8. 回到开发者菜单然后选择Reload JS

小程序开发总结（更新ing）

小程序开发学习过程中的一些总结：

  1. iPhone 和安卓机显示样式不同，需要在 JS 中判断设备类型并针对性解决，如商品促销标签

  2. 不应该频繁使用 setData ，不应该传入大量数据，否则会影响性能

  3. scroll-view 判断是否到达底部，进行数据加载，可以使用 scroll-view 的 bindscrolltolower 属性

  4. line-height 需要比字号大一些，避免整行文字显示不完整，多行文字（多用于商品标题），行高比字号大一点，安卓设备大 3-4rpx，iOS设备大 1-2rpx，文字块的高度，比两倍（或多倍）行高的和小 2rpx

  5. 列表第二列商品标签显示不全问题（如图），原因尚不明确，可能与微信 rpx 计算方式有关系，暂时将外部 card 宽度从 50% 改为定宽解决，应该仍有其他解决方法

  6. wx.getSystemInfo API 非常有用，可以判断平台等等设备信息，微信开发者工具调试中不能模拟，需要真机调试

  7. 商品标签在 iPhone 6/7 和 iPhone 6p/7p 不同尺寸屏幕上，内部文字不垂直居中，可以适当增加1-2rpx 行高

  8. 一些 API 如 createSelectorQuery 是异步操作的，若执行在如 setData 比较大量的数据写入之后时，可能会因为 setData 有性能延迟而发生错误

  9. 小程序内嵌 web-view 刷新方法：小程序自带的原生API只能用于刷新小程序页面，不能刷新 web-view 页面。

    • 最好的方法：使用 wx:if 给 web-view 绑定一个 boolean 变量，利用 if 会刷新 dom 节点的性质，更新 boolean（先false再true）会使 web-view 页面刷新。

      问题：目前没有发现问题

    • 不完美的方法：在需要刷新页面的时候，先将 url 变量变成空字符串，再重新赋值。

      问题：在网页模拟器上，后退会出现url为空时候的错误页，但在iOS真机上正常

    • 不要使用的方法：web-view 页面会在 url 改变的时候刷新，可以通过更改url的方式（如尾部增加一个时间戳作为参数）使页面刷新。

      问题：iOS系统中每次后退时会回到上一个页面，多次跳转时要点击多次后退，使用体验差

  10. 收集 formid 时若需要在内容外包一层 button 标签，应该清除这个 button 的默认样式，避免内部样式错乱

  11. 如何禁止页面滚动：

    不用下拉刷新的方法：

    • 整个布局用<scroll-view>作为根节点，绑定 scroll-view 的 scroll-y 属性<scroll-view scroll-y="false">
    • 样式文件中设置 Page 的 overflow-y 属性值为hidden
    • 样式文件中设置 scroll-view 的 height 属性值为100%

    使用下拉刷新的方法：

    • 在外层添加一个view：<view class="page" style="height: ; overflow: ;"> 然后动态算出设备高度

  12. 【不只是小程序的问题】：JavaScript会自动取消数值类型后面无意义的0，导致解析JSON之后数值位数不一致，建议转换成字符串，或者在数据较少的时候统一toFixed

  13. 屏蔽触摸滑动事件：给元素添加属性 catchtouchmove=”preventTouchMove” 其中preventTouchMove 是空方法

  14. 圆形图片设置 border 可能会在 border 内部出现锯齿，影响样式， 可以在图片外部套一个略大于图片的标签，用背景色的方法解决

  15. 在A页面使用 setTimeout 的时候，方法未执行前如果切换到其他 tab，方法不会被挂起而依然会被执行，开发时需要注意避免这种情况

  16. iPhone 6, iOS 10.3 使用 font-weight: lighter 会出现渲染问题，字体非常细以至于失去本身的颜色，更新：iPhone 6, iOS 12+ 显示正常，推测是iOS版本问题

二叉树与二叉查找树

##树

基本概念

  • 一棵树最上面的节点成为根节点
  • 如果一个节点下面链接多个节点，那么该节点称为父节点，它下面的节点称为子节点
  • 一个节点可以有0个、1个或多个子节点，没有任何子节点的节点称为叶子节点
  • 一个节点到另一个节点的一组边称为路径
  • 每个节点都有一个与之相关的值，称为键值或键

树的层次

根节点是第0层，它的子节点是第1层，子节点的子节点是第2层，以此类推。树的层数就是树的深度

二叉树与二叉查找树

概念

  • 左节点与右节点
  • 二叉查找树（BST）：较小的值保存在左节点，较大的值保存在右节点，这样查找的效率很高，对于数值型和非数值型的数据都是这样

遍历二叉查找树

  • 中序遍历：按节点上的键值，以升序访问查找树上所有节点
  • 前序遍历：先访问根节点，再以同样方式访问左子树和右子树
  • 后序遍历：先访问叶子节点，从左子树到右子树，再到根节点

getElement和query两类获取DOM方式的区别

###获取 DOM 对象有很多种方法，将常用的分为 getElement 和 query 两类

  • getElementById、getElementsByTagName、getElementsByClassName等
  • querySelector、querySelectorAll

####getElementsByTagName()

  • 是document对象特有的函数，只能通过其来调用该方法
  • 返回一个HTMLCollection集合
  • 返回元素的顺序是它们在文档中的顺序
  • 传递给 getElementsByTagName() 方法的字符串可以不区分大小写

####getElementsByClassName()

  • 返回作为 NodeList 对象，代表一个有顺序的节点列表

####querySelector() 与 querySelectorAll()

  • 返回匹配指定 CSS 选择器 元素
  • 该方法只返回匹配指定选择器的第一个元素。如要返回所有匹配元素，需要使用 querySelectorAll() 方法
  • querySelector() 是按css规范来实现的，所以传入的字符串中第一个字符不能是数字

####其他

  • query方法出来的元素及元素数组是静态的，不会随着文档操作而改变
  • getElement这类方法选出的元素是动态的
  • getElement这类方法性能比较好，query则比较方便

CSS3中的transform详解

转载自：https://www.cnblogs.com/afighter/p/5726888.html

一、变形-旋转 ratate() 函数

通过指定的角度参数使元素相对原点进行旋转。它主要在二维空间内进行操作，设置一个角度值，用来指定旋转的幅度。如果这个值为正值，元素相对原点中心顺时针旋转；如果这个值为负值，元素相对原点中心逆时针旋转。

例：transform:rotate(-20deg)

二、变形- 扭曲 skew()函数

扭曲skew()函数能够让元素倾斜显示。它可以将一个对象以其中心位置围绕着X轴和Y轴按照一定的角度倾斜。这与rotate()函数的旋转不同，rotate()函数只是旋转，而不会改变

元素的形状。skew()函数不会旋转，而只会改变元素的形状。

  • 一个参数：表示水平方向的倾斜角度；
  • 两个参数：第一个参数表示水平方向的倾斜角度，第二个参数表示垂直方向的倾斜角度

分3种情况

transform:skewX(30deg)

transform:skewY(10deg)

transform:skew(-45deg)

三、变形-缩放 scale() 函数

让元素根据中心原点对对象进行缩放。 scale()的取值默认的值为1，当值设置为0.01到0.99之间时，作用使一个元素缩小；而任何大于或等于1.01的值，作用是让元素放大。

也分3种情况：

transform:scale(x,y) 使元素水平方向和垂直方向同时缩放

transform:scale(x) X轴缩放

transform:scale(y) Y轴缩放

四、变形-位移 translate() 函数

translate()函数可以将元素向指定的方向移动，类似于position中的relative。或以简单的理解为，使用translate()函数，可以把元素从原来的位置移动，而不影响在X、Y轴上

的任何Web组件。

分为三种情况：

transform:translate(x,y) 使元素水平方向和垂直方向同时移动

transform:translate(x) 使元素水平方向移动

transform:translate(y) 使元素垂直方向移动

五、变形-原点 transform-origin

任何一个元素都有一个中心点，默认情况之下，其中心点是居于元素X轴和Y轴的50%处。

在没有重置transform-origin改变元素原点位置的情况下，CSS变形进行的旋转、位移、缩放，扭曲等操作都是以元素自己中心位置进行变形。但很多时候，我们可以通过

transform-origin 来对元素进行原点位置改变，使元素原点不在元素的中心位置，以达到需要的原点位置。

transform-origin 取值和元素设置背景中的background-position取值类似(也可以是具体的像素值)。

Mac急救办法

SHIFT+CTRL+OPTION+POWER十秒钟：重置系统管理控制器（SMC）

  • 关闭电脑。
  • 将 MagSafe 电源适配器连接到电源和 Mac（如果尚未连接的话）。
  • 在内建键盘上，同时按下（左侧）Shift-Control-Option 键和电源按钮。
  • 同时松开所有键和电源按钮。
  • 按电源按钮打开电脑。

注：重置 SMC 时，MagSafe 电源适配器上的 LED 指示灯可能会更改状态或暂时关闭。

在风扇、电源指示灯、系统性能、视频出现某些异常时候，可以通过上述方法进行重置来解决。具体的症状判断和说明可以参考”基于 Intel 的 Mac：重置系统管理控制器（SMC）“。

OPTION+CMD+R+P+POWER：重置NVRAM 或 PRAM

Mac 会将某些设置储存在特殊内存区域中，而且即使关机这些设置也不会丢失（除非存在电池问题）。 在基于 Intel 的 Mac 上，存储位置是称为 NVRAM 的内存；而在基于 PowerPC 的 Mac 上，存储位置则是称为 PRAM 的内存。操作步骤为：

  • 关闭电脑。
  • 在键盘上找到以下按键：Command、Option、P 和 R。您需要在步骤 4 中同时按下这些键。
  • 启动电脑。
  • 按住 Command-Option-P-R 键。必须在出现灰屏前按下此组合键。
  • 按住上述键，直到电脑重新启动，您会再次听到启动声。
  • 松开这些键。

在NVRAM和PRAM中保存着很多硬件的配置，并且不会因为关机而丢失。详细的信息可以参考”重置 Mac 的 PRAM 和 NVRAM“，”何时要重置 NVRAM 或 PRAM“，”Mac OS X：PRAM 中存储的是什么“。

设计模式

思考：

  • 我们为什么要设计/抽象，它的本质是什么
  • JavaScript 有什么特点，什么样的设计模式适合这门语言
  • 了解“逻辑”背后的信息比如：满足什么需求？解决什么问题？变化会多频繁？有什么痛点？

（世面上JS设计模式的书基本都在扯淡，很多设计模式是不适合JS的）

模式1：Behavior

  • 在交互需求出发设计一组“动作行为”
  • 在主体对象上应用和扩展这些行为
  • 为什么这么做？

实例1：图片预览（可切换到上一张下一张图片）

分析：Behavior 模式：自定义 preview 行为

思考与练习：

  1. 如果图片要动态加载，怎么修改 setPreviewBehavior?
  2. 如果要兼容旧的浏览器，如IE7，如何处理？
  3. 为上一张、下一张图片切换添加动画

实例2：选取图片，自定义 setSelectionBehavior 行为

模式2：Adapter

  • 针对不同底层实现提供统一的外部接口
  • 让业务层面的代码可以适应不同的底层实现

即针对不同的状况进行判断

Behavior + Adaptor

Web开发最核心的就是事件驱动，事件是一个异步模型，所以observer（观察者模式）很重要

Observer 用于

  • 同步多个对象的状态变化
  • 解耦各个对象之间的依赖

  • 将命令传递给所有订阅者
  • 与所有订阅者之间维持依赖关系
  • 从而让订阅者彼此之间相互独立

  • 对已有组件/库/功能的组合使用

模式9：Aspect（切面）

  • 对代码的执行过程进行监测和修改
