module.exports与exports, export与export default之间的关系和区别

首先我们要明白一个前提，CommonJS模块规范和ES6模块规范完全是两种不同的概念。

CommonJS模块规范

Node应用由模块组成，采用CommonJS模块规范。

根据这个规范，每个文件就是一个模块，有自己的作用域。在一个文件里面定义的变量、函数、类，都是私有的，对其他文件不可见。

CommonJS规范规定，每个模块内部，module变量代表当前模块。这个变量是一个对象，它的exports属性（即module.exports）是对外的接口。加载某个模块，其实是加载该模块的module.exports属性。

exports 与 module.exports

为了方便，Node为每个模块提供一个exports变量，指向module.exports。这等同在每个模块头部，有一行这样的命令。

于是我们可以直接在 exports 对象上添加方法，表示对外输出的接口，如同在module.exports上添加一样。注意，不能直接将exports变量指向一个值，因为这样等于切断了exports与module.exports的联系。

ES6模块规范

不同于CommonJS，ES6使用 export 和 import 来导出、导入模块。

需要特别注意的是，export命令规定的是对外的接口，必须与模块内部的变量建立一一对应关系。

Git的常用命令及使用

基本命令

  1. git init初始化
  2. git add <file> 添加文件，把要提交的所有修改放到暂存区，每次修改都要add一下
  3. git commit -m 提交所有修改，后面加改动记录，commit 之后暂存区就清空了
  4. git status 查看工作区状态，看看是否有文件修改过
  5. git diff 查看difference，看看修改内容

时空穿梭

版本回退

git reset --hard commit_id 在版本的历史之间穿梭

  • HEAD 代表当前版本， HEAD^ 是当前版本的上个版本，可行的写法 HEAD^^ HEAD~100
  • commit_id 是一次提交产生的id
    git log 查看提交历史，以便确定要回退到曾经的哪个版本 git reflog 回退版本之后，未来版本不会出现在日志中，使用这个命令查看命令历史，以便确定要回到未来的哪个版本
  • --hard 意思是不保留恢复的版本之后的所有修改，默认 --soft 意思是保留
  • reset 之后，如果想覆盖远程分支，执行 push -f，否则会一直提示版本落后

撤销修改

命令git checkout -- readme.txt意思就是，把readme.txt文件在工作区的修改全部撤销，这里有两种情况：

  • 一种是readme.txt自修改后还没有被放到暂存区，现在，撤销修改就回到和版本库一模一样的状态；
  • 一种是readme.txt已经添加到暂存区后，又作了修改，现在，撤销修改就回到添加到暂存区后的状态。 总之，就是让这个文件回到最近一次git commit或git add时的状态
    git checkout -- file命令中的\--很重要，没有\--，就变成了“切换到另一个分支”的命令 命令git reset HEAD file 除了回退版本，还可以把暂存区的修改撤销掉（unstage），重新放回工作区

删除文件

一般情况下，你通常直接在文件管理器中或者用rm命令把没用的文件删了，Git知道你删除了文件，因此，工作区和版本库就不一致了。

  • 一是确实要从版本库中删除该文件，那就用命令git rm删掉，并且git commit
$ git rm test.txt
$ git commit -m "remove test.txt"
  • 另一种情况是删错了，因为版本库里还有，所以可以很轻松地把误删的文件恢复到最新版本
$ git checkout -- test.txt

远程仓库

前期设置

  1. 创建SSH Key：在用户主目录下，看看有没有.ssh目录以及这个目录下有没有id_rsa和id_rsa.pub这两个文件，如果已经有了，可直接跳到下一步。如果没有，打开Shell（Windows下打开Git Bash），创建SSH Key：
$ ssh-keygen -t rsa -C "youremail@example.com"
  1. 登陆GitHub，打开“Account settings”，“SSH Keys”页面： 然后，点“Add SSH Key”，填上任意Title，在Key文本框里粘贴id_rsa.pub文件的内容：点“Add Key”，你就应该看到已经添加的Key

添加仓库

  1. 关联一个远程库
$ git remote add origin git@github.com:username/reponame.git
  1. 删除已有的远程库
$ git remote rm origin
  1. 推送暂存库内容到远程库的 master 分支
$ git push -u origin master

第一次推送的时候加上-u参数，Git不但会把本地的master分支内容推送到远程新的master分支，还会把本地的master分支和远程的master分支关联起来，在以后的推送或者拉取时就可以简化命令

  1. 查看当前的远程库信息
git remote -v
  1. 用命令git clone 把远程库克隆到本地库
$ git clone git@github.com:username/reponame.git
  1. SSH警告： 第一次使用Git的clone或者push命令连接GitHub时，会得到一个警告：输入yes回车即可，Git把GitHub的Key添加到本机的一个信任列表里

分支管理

基本操作

1. 查看分支：`git branch`
2. 创建分支：`git branch <name>`
3. 删除本地分支：git branch -D 分支名
